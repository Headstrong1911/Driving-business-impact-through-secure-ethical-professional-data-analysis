PROJECT EVIDENCE 1
This report aims to conduct a root cause analysis of wrong item picking. 
This project focuses on identifying the root causes of wrong item picking within warehouse operations, wrong item picking occurs when an incorrect product is selected during the order fulfilment process, resulting to customer complaints, product returns, additionally delivery cost and operational inefficiencies. 
The issue has become increasingly important within the business due to a rise in picking errors over recent months. These errors not only affect customer satisfaction but also increase operational costs through rework, refunds and delays. The purpose of this analysis is to investigate the underlying causes of the issue and identify solutions that can reduce future errors and improve operational performance. 
This project supports the business by using secure, ethical, and professional data analysis to understand the true cause of the problem rather than relying on assumptions. By identifying the root cause, the business can implement preventative measures that reduce recurrence and improve efficiency

STRUCTURED AND UNSTRUCTURED DATA (K5)
This project used both structured and unstructured data to investigate the cause of wrong item picking. 
Structured data refers to data that is organised into predefined formats such as tables, rows and columns. Examples used within this project included:
•	Picking log
•	Order records
•	Inventory reports 
•	Return data
•	Product location data
Structured data is easier to sort, filter and analyse because it follows a consistent format. 
Unstructured data refers to data that does not follow a predefined format and is often more difficult to analyse. Examples are:
•	Employee feedback
•	Customer complaints
•	Warehouse observations
The main difference between structured and unstructured data is how the information is organized and processed. Structured data is easier to analyse using systems and software, whereas unstructured data often requires interpretation to identify trends and insights. 
One challenge of working with unstructured data is inconsistency, as a different employees may describe issues differently. To mitigate this challenge, common themes and recurring issues were identified manually through reviewing feedback and operational observations
Using both structured and unstructured data provided a more complete understanding of the operational issues contributing to wrong item picking.

IDENTIFYING THE ROOT CAUSE AND PREVENTING RECURRENCE (B5)
The Problem investigated within this project was the increasing number of wrong item picking errors within the warehouse operations. Initially, the exact cause of the issue was unknown, requiring a structured investigation process to identify the true root cause. 
To identify the issue, I reviewed operational data, analysed picking trends, compared error rate across warehouse zones and gathered feedback from the warehouse staff. I also used the 5 whys root cause analysis technique to investigate the underlying cause rather than focusing only on the surface – level symptoms. 
The Analysis identified several contributing factors, which are:
•	Similar looking products stored close together
•	Poor shelf labelling
•	Inconsistent barcode scanning procedures 
•	Limited verification checks before dispatch
•	Increased picking pressure during busy periods. 
The root cause of the issue was primarily linked to poor inventory location management and inconsistent operational procedures. 
To prevent recurrences, several solutions were recommended which includes:
•	Improving warehouse labelling
•	Introducing mandatory barcode scanning
•	Separating similar products 
•	Conducting regular inventory audits
•	Providing refresher training for employees
These solutions are designed to reduce future picking errors and improve operational consistency across warehouse processes. By identifying the true root cause and implementing preventative measures, the business can definitely improve efficiency, also reduce costs and enhance customer satisfaction in long term.
